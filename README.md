## Firebase Setup

- **Firebase Project:** `fae-web-presence`
- **Console:** https://console.firebase.google.com/project/fae-web-presence/overview
- **Hosting:** https://console.firebase.google.com/project/fae-web-presence/hosting/sites

### How Multi-Site Hosting Works

Each site has a **deploy target** (defined in `.firebaserc`) that maps to a **Firebase site ID**. The `firebase.json` maps each target to a local directory (`public` field). When you deploy, Firebase reads these mappings and pushes each directory to its respective site.

Note: `thefae.com` uses the Firebase site ID `thefae-site` (not `thefae-com`) because that ID was reserved by another project.

## Development

### Prerequisites

```bash
npm install -g firebase-tools
firebase login
```

### Deploy All Sites

```bash
firebase deploy --only hosting
```

### Deploy a Single Site

```bash
firebase deploy --only hosting:thefae-com
firebase deploy --only hosting:myfae-ai
firebase deploy --only hosting:meetfae-ai
# etc.
```

The target name after `hosting:` matches the keys in `firebase.json` (e.g., `thefae-com`, `myfae-ai`).

### Preview Before Deploying

```bash
firebase hosting:channel:deploy preview --only thefae-com
```

### Adding a New Site

1. Create the Firebase hosting site:
   ```bash
   firebase hosting:sites:create new-site-id --project fae-web-presence
   ```
2. Add a deploy target in `.firebaserc` under `targets.fae-web-presence.hosting`
3. Add a hosting entry in `firebase.json` with the target and public directory
4. Create the directory with an `index.html`
5. Add the custom domain in the Firebase console
6. Set DNS records (A records to `151.101.1.195` and `151.101.65.195`)

## DNS Configuration

All domains use the same A records pointing to Firebase Hosting:

| Type | Host | Value |
|------|------|-------|
| A | @ | `151.101.1.195` |
| A | @ | `151.101.65.195` |

For www subdomains, add a CNAME:

| Type | Host | Value |
|------|------|-------|
| CNAME | www | `<firebase-site-id>.web.app` |

Custom domains also require a TXT verification record generated by Firebase when you add the domain in the console.

DNS is managed via GoDaddy.
